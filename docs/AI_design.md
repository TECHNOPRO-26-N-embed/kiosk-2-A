## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. 商品マスタ（最大10件）を初期化する
2. メインループ開始
3. カテゴリ一覧を表示し、カテゴリ番号を入力させる
4. 入力が不正ならエラーメッセージを表示して3に戻る
5. 選択カテゴリの商品一覧を表示する
6. 商品番号と数量を入力させ、購入リストに追加する
7. 「追加購入する/会計へ進む」を選ばせる
8. 追加購入なら3へ戻る
9. 会計へ進む場合、合計金額・税率別内訳を計算する
10. レシートを表示する
11. 必要に応じてお客様呼び出し機能を実行する
12. 取引ログをCSVへ追記する
13. 次の顧客処理のため購入リストを初期化し、3へ戻る
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: 購入処理

```
1. カテゴリ番号を受け取る
2. 対象カテゴリの商品配列を表示する
3. 商品番号を受け取る
4. 数量を受け取る（1以上の整数）
5. 商品番号・数量の妥当性をチェックする
6. 購入リストへ追加する
7. 追加購入フラグがONなら1へ戻る
8. 購入終了なら会計処理へ進む
```

#### 機能2: ログ保存処理

```
1. 取引IDを生成する（日付+連番）
2. 購入リストを先頭から順に処理する
3. 各商品について小計・税額・合計を計算する
4. CSV1行文字列を作成する
5. data/sales_log.csv を追記モードで開く
6. ファイルが空ならヘッダを書き込む
7. 明細行を書き込む
8. 全明細の書き込み後にファイルを閉じる
```

#### 機能5: レシート表示機能

```
1. 購入リストから商品明細を作成する
2. 合計金額を計算する
3. 税率別内訳を計算する
4. レシート内容を画面に表示する
```

#### 機能6: お客様呼び出し機能

```
1. エラーやトラブルが発生した場合に呼び出しフラグを設定する
2. 呼び出しフラグが有効な場合、画面に「店員を呼び出してください」と表示する
3. 呼び出し状況をログに記録する
```

### 4.3 エラーハンドリング（想定される異常系）

> ユーザーが「想定外の操作」をした場合にどうするか検討してください。

| 異常系シナリオ | 対処方針 |
| :--- | :--- |
| 数字以外の文字が入力された場合 | 入力を破棄して「数値を入力してください」と表示し、同じ入力箇所で再入力させる。 |
| 存在しないカテゴリ・商品番号が入力された場合 | 有効範囲を表示して再入力させる。 |
| 数量に0以下が入力された場合 | 1以上の整数を要求して再入力させる。 |
| 金額が不足している場合 | 不足額を表示し、受取金額を再入力させる。 |
| CSVファイルが開けない場合 | エラーメッセージを表示し、標準出力への簡易ログ表示に切り替える。 |
| 商品が未選択のまま会計に進んだ場合 | 「商品を1つ以上選択してください」と表示し、商品選択へ戻す。 |

## 5. 備考・メモ

> 設計中に気になった点や、明日の実装に向けた注意点があれば記録してください。

* 税計算は「商品ごとに税計算して合算」か「税率別に合算して税計算」かを実装前に統一する。
* 商品マスタは配列で保持し、カテゴリID・税率・価格を構造体で管理する。
* 支払方法は列挙値（例: cash, credit, emoney）で管理し、今回はcashのみ実装する。
* CSV保存先の data ディレクトリが存在しない場合の作成処理を検討する。
