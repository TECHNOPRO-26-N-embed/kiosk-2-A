## 3. 基本設計

### 3.1 画面遷移図
> テキスト形式で画面の状態遷移を記述してください。
> 例: `メインメニュー → [1] → 購入画面 → [確定] → 完了画面 → メインメニュー`

```
起動
	↓
カテゴリ選択画面
	↓
商品一覧（最大10種）から選択・数量入力
	↓
購入リスト確認（追加購入 or 会計へ）
	↓
合計金額確認
	↓
受取金額入力・会計確定
	↓
CSV保存・完了画面
	↓
カテゴリ選択画面へ戻る
```

### 3.2 データの洗い出し

> このシステムが**覚えておくべき情報**を洗い出してください。
> まずは「何の情報が必要か」を自然言語で書き出せばOKです。
> （C言語での型や構造体の定義は、明日の詳細設計で行います）

| 情報の名前 | 内容・説明 | 例 |
| :--- | :--- | :--- |
| 取引ID | 1会計を一意に識別するID | 20260519-0001 |
| 購入日時 | 会計が確定した日時 | 2026/05/19 10:15:20 |
| 商品ID | 商品を一意に識別する番号 | 3 |
| 商品名 | 各商品の名称 | トマト |
| 商品カテゴリ | 商品の分類 | 野菜 |
| 単価 | 商品1個あたりの価格 | 120 |
| 数量 | 購入した個数 | 2 |
| 税率 | 商品に適用する税率 | 8 |
| 小計 | 税抜または税込の基準金額（実装で統一） | 240 |
| 税額 | 税率から計算した税金 | 19 |
| 合計金額 | 支払対象の合計金額 | 259 |
| 受取金額 | 客から受け取った金額 | 500 |
| お釣り | 受取金額 - 合計金額 | 241 |
