## 2. 要求仕様書（顧客ヒアリング結果）

### 2.1 顧客の要望（ヒアリングメモ）

> ここに、顧客（講師）へのヒアリングで得た要望を箇条書きで記録してください。

* 商品をカテゴリで選べる　野菜、果物、などから選択する形
* 商品種類が最大1500種類
* 税率別(8%,10%)
* 支払方法　複数あればありがたい実装できなくても拡張できれば
* CSVにログを保存する
### 2.2 機能要件

| 機能ID | 機能名 | 概要 |
| :---: | :--- | :--- |
| F01 |商品選択機能 |数字の入力によってカテゴライズされる |
| F02 |税率判別機能 |店内利用・テイクアウトで税率を変える |
| F03 |お釣り計算機能 |支払われたお金からお釣りを計算する |
| F04 |レシート発行機能 |レシートとして購入したものをリストにする |

### 3.2 データの洗い出し

> このシステムが**覚えておくべき情報**を洗い出してください。
> まずは「何の情報が必要か」を自然言語で書き出せばOKです。
> （C言語での型や構造体の定義は、明日の詳細設計で行います）

| 情報の名前 | 内容・説明 | 例 |
| :--- | :--- | :--- |
|商品名|各商品の名称 |うまい棒 |
|単価 |値段 |10円 |
|商品カテゴリ |商品を見つけやすくするための大まかな区分 |やさい |
|在庫数 |店内にある在庫数 |100個 |
|税区分 |店内利用かテイクアウトアウトか |8% |
|鮮度 |安全に提供できる時間 |賞味、消費期限 |
|顧客情報 |カードに登録されている顧客情報 |ポイント、会員カード |
|日時|購入日 |05/15　05:15|
|数量|同一商品の個数|10個|
