## 4. 詳細設計（アルゴリズム）

### 4.1 メインループ

> メニュー表示→入力受付→処理分岐→ループの流れを記述してください。
> フローチャートまたは疑似コード（箇条書き）で記述してください。

```
1. メインメニューで入力
   - 1の場合: 商品カテゴリ
   - 2の場合: 店員を呼び出す
   - 3の場合: 従業員モード(強制終了)
   - 4の場合: 商品検索
2. メインメニューに戻る（1へ）
```

### 4.2 個別機能のアルゴリズム

> 各機能（購入処理、ログ保存、ログ表示など）の処理フローを記述してください。

#### 機能1: 商品選択

```
1.対応する商品番号を受け取る
2.在庫を確認する
3.選択された商品の名前と値段を受け取る
4.選択された商品の名前と値段を表示する
5.商品の合計値段を表示する
上記を会計ボタンが押されるまでループ
```

#### 機能2: 計算

```
1.総合金額受け取り
2.支払った金額受け取り
3.(支払った金額) – (総合金額受け取り)
4.return value
5.ポイントカードで進む
```

#### 機能3: レシート発行
```
1.レシートを発行するかしないか表示(1→発行、0→発行しない)
2.ユーザーに入力してもらう(scanf)
3.入力によって分岐
　3-1…発行するならかごに入った商品の商品名、単価、(個数？)を順に出力し、最後に合計を出力する
　3-2…発行しなければ、終了しメインメニュー画面に戻る
　3-3…数字以外の入力は[数字を入力してください]と表示して２．にもどる
```

#### 機能4: 商品検索機能
```
1 検索ページのボタンを作る
2 検索ページに飛ぶ機能
3 検索の欄に打ち込まれた商品名を照合する機能
（エラー出力:商品が見つかりません）
（完璧に商品名を検索するのは難しいかもしれないので、適合する文字が多い商品を表示する）
4 商品選択機能
5 商品ページに飛ぶ機能
6 商品の個数を選択する機能
7 検索ページに戻る機能
```

#### 機能5: 店員呼び出し機能
```
1 利用者が「店員呼び出し」ボタンを押す
  2. 現在の端末番号を取得する
  3. 現在の会計番号を取得する
  4. 呼び出し理由を選択させる
  5. 呼び出し理由が入力されているか確認する
  6. すでに呼び出し中か確認する
  7. 呼び出し中でなければ呼び出し番号を発行する
  8. 呼び出し情報を保存する
  9.呼び出し情報を「未対応」にする
  10. 店員側の画面に通知ボタンを送る
  11. 利用者画面に「店員を呼び出しています」を表示する
```
